The Astro codebase uses logos and icons in several distinct contexts. Here is a breakdown of each.

---

## 1. Site Favicon

The standard Astro project favicon is an **SVG file** placed in the `public/` directory.

**File:** `public/favicon.svg`
**Format:** SVG with `viewBox="0 0 128 128"` (or `0 0 36 36` in some fixtures)
**Reference in HTML:**

```html
<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
```

This pattern appears consistently across Astro's own fixture projects. [1](#0-0) 

Some older fixtures use `.ico` format instead:

```html
<link rel="icon" type="image/x-icon" href="/favicon.ico" />
``` [2](#0-1) 

The default Astro favicon SVG supports dark mode via a CSS media query inside the SVG: [3](#0-2) 

To **change the favicon**, replace `public/favicon.svg` with your own SVG (or other image format), and update the `<link rel="icon">` tag in your layout's `<head>`.

---

## 2. VS Code Language Extension Icons

The Astro VS Code extension ships two icon variants for the `.astro` file type:

| File | Theme | viewBox |
|---|---|---|
| `packages/language-tools/vscode/assets/lang-icon-dark.svg` | Dark | `0 0 24 24` |
| `packages/language-tools/vscode/assets/lang-icon-light.svg` | Light | `0 0 24 24` |

Both are **24×24 SVG** files using the Astro flame logo with a gradient tail (colors `#D83333` → `#F041FF`). [4](#0-3) 

---

## 3. Dev Toolbar App Icons

The Dev Toolbar renders app icons at **20×20px** in the toolbar bar. [5](#0-4) 

The `icon` field on a `DevToolbarApp` is typed as `Icon`:

```ts
type DevToolbarAppMeta = {
  id: string;
  name: string;
  icon?: Icon;
};
``` [6](#0-5) 

`Icon` is defined as either a **predefined icon name** (a key of the built-in `icons` object) or a **raw SVG string**:

```ts
export type DefinedIcon = keyof typeof icons;
export type Icon = DefinedIcon | (string & NonNullable<unknown>);
``` [7](#0-6) 

### Built-in icon names

The full list of predefined icon names is defined in `packages/astro/src/runtime/client/dev-toolbar/ui-library/icons.ts`: [8](#0-7) 

Key names include: `astro:logo`, `warning`, `arrow-down`, `bug`, `check-circle`, `gear`, `lightbulb`, `file-search`, `star`, `checkmark`, `dots-three`, `copy`, `compress`, `grid`, `puzzle`, `approveUser`, `checkCircle`, `resizeImage`, `searchFile`, `image`, `robot`, `sitemap`, `gauge`, `person-arms-spread`, `arrow-left`, `houston-detective`.

### How built-in apps use icons

- `astro:home` (Menu) uses the predefined name `'astro:logo'` — the Astro flame, `viewBox="0 0 85 107"`
- `astro:settings` uses `'gear'`
- `astro:xray` (Inspect) uses a raw SVG string with `viewBox="0 0 24 24"`
- `astro:audit` uses a raw SVG string with `viewBox="0 1 20 16"` [9](#0-8) 

### How to set a custom app icon

When building a custom Dev Toolbar app, set `icon` to either a predefined name or an inline SVG string:

```ts
// Using a predefined name
export default {
  id: 'my-app',
  name: 'My App',
  icon: 'gear',
  init(canvas) { ... }
}

// Using a raw SVG string
export default {
  id: 'my-app',
  name: 'My App',
  icon: '<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">...</svg>',
  init(canvas) { ... }
}
```

The `getAppIcon()` function handles both cases — if the string is a known key it looks it up, otherwise it passes the raw SVG through directly: [10](#0-9)

## 4. SVG as a Component (Asset Import)

Astro also supports importing `.svg` files directly as Astro components (stable since v5.6):

```astro
---
import Logo from './path/to/logo.svg';
---
<Logo width={64} height={64} fill="currentColor" />
```

The `SvgComponent` type can be imported from `astro/types` for type safety:

```ts
import type { SvgComponent } from 'astro/types';
``` [11](#0-10)
