## 1. Executive Summary & Core Identity

**HMR (همر)** is an AI-powered **mobile-hardware advisor chatbot** engineered specifically for the
Persian-speaking Iranian market. It bridges the gap between complex consumer electronics and everyday
users by providing clear, data-driven guidance on buying, diagnosing, and maintaining mobile devices.

A foundational tenet of the project is its operating philosophy: **HMR acts strictly as a technical
advisor, not a final decision-maker.** It empowers users with recommendations while leaving purchasing
or repair decisions entirely in their hands. The service is delivered across a **dual-platform
ecosystem** — a responsive web application and a native Android app.

| | |
|---|---|
| **Name** | HMR (همر) |
| **Nature** | AI chatbot specialized in mobile technology |
| **Role** | Advisor — not a decision-maker |
| **Platforms** | Web application + Android application |
| **Audience** | Persian-speaking users (Iranian market) |
| **Language rule** | End-user output in **Persian**; docs & code in English |
| **Honesty rule** | Never fabricate specs or prices — **live sources only** |

## 3. Technical Architecture & System Components

The project uses a decoupled, modern **Jamstack + micro-agent** architecture designed for performance,
scalability, and ease of maintenance.

| Component | Technology | Specifications & Configuration |
| :--- | :--- | :--- |
| **AI Chatbot Engine** | **FlowiseAI** | Agentflows v2 / Multi-Agents. Active chatflow: **`HMR-Agentflows-v2`**. Model routing via OpenRouter: LLM `gemini-3.1-flash-lite`, embeddings `openai/text-embedding-3-large`, reranker `google/semantic-ranker-default-004`. |
| **Live Web Search** | **SearXNG** | Self-hosted metasearch (Docker container `docker-searxng-1`) powering the chatbot's live search tool — the backbone of the *live-price-only* rule. |
| **Backend Server (VPS)** | **Docker / Ubuntu** | Containerized on Ubuntu 24.04. Nuremberg · 2-Core CPU · 4 GB RAM · 40 GB storage · 1 dedicated IP (`91.107.159.48`). |
| **Web Application** | **Astro** | Modern SSG frontend, deployed globally via **Cloudflare** (Workers / Pages). |
| **Android Application** | **Flutter** | Cross-platform build for Android. Auth via **Google Sign-In (Firebase + Google Cloud)** — Project ID `ir-hmrbot-app`, Project Number `829078792642`. |
| **System Monitoring** | **UptimeRobot** | 24/7 uptime and health tracking across critical endpoints. |

<Aside type="note" title="Security posture">
The FlowiseAI prediction endpoint is **private** (`isPublic=0`). The `Authorization: Bearer` token is
injected **server-side by nginx** and never ships in the public web bundle. Nightly encrypted backups of
the Flowise data (`/root/.flowise`) run to Telegram, plus a nightly state snapshot to the `HMR-VPS` repo.
</Aside>

## 6. Reliability & Health Monitoring

To ensure continuous availability for Persian-speaking users, system uptime is tracked 24/7 via
**UptimeRobot**. Monitoring evaluates three primary endpoints to verify full-stack health:

- **`hmrbot.com`** — frontend availability and Cloudflare routing.
- **`hmrbot.com/api/health`** — core API gateway and internal response status.
- **`srv.hmrbot.com`** — VPS, FlowiseAI engine, and LLM processing.
