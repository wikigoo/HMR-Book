class="astro-2sm4b3ln">Icons</h1></div></div><div class="content-panel astro-oil7k7jf"><div class="sl-container astro-oil7k7jf"><div class="sl-markdown-content"><p><img src="https://docs-assets.developer.apple.com/published/712733b3cdd2bab5a2c326c50488aeb3/foundations-icons-intro~dark%402x.png" alt="A sketch of the Command key icon. The image is overlaid with rectangular and circular grid lines and is tinted yellow to subtly reflect the yellow in the original six-color Apple logo."></p>
<p>Apps and games use a variety of simple icons to help people understand the items, actions, and modes they can choose. Unlike <a href="https://developer.apple.com/design/human-interface-guidelines/app-icons">app icons</a>, which can use rich visual details like shading, texturing, and highlighting to evoke the app’s personality, an <em>interface icon</em> typically uses streamlined shapes and touches of color to communicate a straightforward idea.</p>
<p>You can design interface icons — also called <em>glyphs</em> — or you can choose symbols from the SF Symbols app, using them as-is or customizing them to suit your needs. Both interface icons and symbols use black and clear colors to define their shapes; the system can apply other colors to the black areas in each image. For guidance, see <a href="https://developer.apple.com/design/human-interface-guidelines/sf-symbols">SF Symbols</a>.</p>
<div class="sl-heading-wrapper level-h2"><h2 id="best-practices">Best practices</h2><a class="sl-anchor-link" href="#best-practices"><span aria-hidden="true" class="sl-anchor-icon"><svg width="16" height="16" viewBox="0 0 24 24" fill="currentColor"><path d="m12.11 15.39-3.88 3.88a2.52 2.52 0 0 1-3.5 0 2.47 2.47 0 0 1 0-3.5l3.88-3.88a1 1 0 1 0-1.42-1.42l-3.88 3.89a4.48 4.48 0 0 0 6.33 6.33l3.89-3.88a1 1 0 0 0-1.42-1.42m8.58-12.08a4.49 4.49 0 0 0-6.33 0l-3.89 3.88a1 1 0 1 0 1.42 1.42l3.88-3.88a2.52 2.52 0 0 1 3.5 0 2.47 2.47 0 0 1 0 3.5l-3.88 3.88a1 1 0 0 0 0 1.42 1 1 0 0 0 1.42 0l3.88-3.89a4.49 4.49 0 0 0 0-6.33M8.83 15.17a1 1 0 0 0 .71.29 1 1 0 0 0 .71-.29l4.92-4.92a1 1 0 1 0-1.42-1.42l-4.92 4.92a1 1 0 0 0 0 1.42"></path></svg></span><span class="sr-only" data-pagefind-ignore>Section titled “Best practices”</span></a></div>
<p><strong>Create a recognizable, highly simplified design.</strong> Too many details can make an interface icon confusing or unreadable. Strive for a simple, universal design that most people will recognize quickly. In general, icons work best when they use familiar visual metaphors that are directly related to the actions they initiate or content they represent.</p>
<p><strong>Maintain visual consistency across all interface icons in your app.</strong> Whether you use only custom icons or mix custom and system-provided ones, all interface icons in your app need to use a consistent size, level of detail, stroke thickness (or weight), and perspective. Depending on the visual weight of an icon, you may need to adjust its dimensions to ensure that it appears visually consistent with other icons.</p>
<p><img src="https://docs-assets.developer.apple.com/published/cd23869c7e26f0f5588e065df7f4a244/custom-icon-sizes~dark%402x.png" alt="Diagram of four glyphs in a row. From the left, the glyphs are a camera, a heart, an envelope, and an alarm clock. Two horizontal dashed lines show the bottom and top boundaries of the row and a horizontal red line shows the midpoint. All four glyphs are solid black; some include interior detail lines in white. Parts of the alarm clock extend above the top dashed line because its lighter visual weight requires greater height to achieve balance with the other glyphs."></p>
<p>To help achieve visual consistency, adjust individual icon sizes as necessary…</p>
<p><img src="https://docs-assets.developer.apple.com/published/fb654ccd9891f6cdd920ae36452da344/custom-icon-line-weights~dark%402x.png" alt="Diagram of the same four glyphs shown above and the same horizontal dashed lines at top and bottom and horizontal red line through the middle. In this diagram, all four glyphs are solid gray; the interior detail lines are black to emphasize that all lines use the same weight."></p>
<p>…and use the same stroke weight in every icon.</p>
<p><strong>In general, match the weights of interface icons and adjacent text.</strong> Unless you want to emphasize either the icons or the text, using the same weight for both gives your content a consistent appearance and level of emphasis.</p>
<p><strong>If necessary, add padding to a custom interface icon to achieve optical alignment.</strong> Some icons — especially asymmetric ones — can look unbalanced when you center them geometrically instead of optically. For example, the download icon shown below has more visual weight on the bottom than on the top, which can make it look too low if it’s geometrically centered.</p>
<p><img src="https://docs-assets.developer.apple.com/published/3160de55e76ebaecd7787b43d6105ab9/asymmetric-glyph~dark%402x.png" alt="Two images of a white arrow that points down to a white horizontal line segment within a black disk. The image on the right includes two horizontal pink bars — one between the top of the glyph and the top of the disk and the other between the bottom of the glyph and the bottom of the disk — that show the glyph is geometrically centered within the disk."></p>
<p>An asymmetric icon can look off center even though it’s not.</p>
<p>In such cases, you can slightly adjust the position of the icon until it’s optically centered. When you create an asset that includes your adjustments as padding around an interface icon (as shown below on the right), you can optically center the icon by geometrically centering the asset.</p>
<p><img src="https://docs-assets.developer.apple.com/published/52b1add5b34d3ce3cb314922dbcd76b5/asymmetric-glyph-optically-centered~dark%402x.png" alt="Two images of a white arrow that points down to a white horizontal line segment within a black disk. The image on the left includes the two horizontal pink bars in the same locations as in the previous illustration, but the glyph has been moved up by a few pixels. The image on the right includes a pink rectangle overlaid on top of the glyph to represent a padding area, which includes the extra pixels below the glyph."></p>
<p>Moving the icon a few pixels higher optically centers it; including the pixels in padding simplifies centering.</p>
<p>Adjustments for optical centering are typically very small, but they can have a big impact on your app’s appearance.</p>
<p><img src="https://docs-assets.developer.apple.com/published/fec10f5a84294fe9be0a24aab37766a5/asymmetric-glyph-before-and-after~dark%402x.png" alt="Two images of a white arrow that points down to a white horizontal line segment within a black disk. The glyph on the left is geometrically centered and the one on the right is optically centered."></p>
<p>Before optical centering (left) and after optical centering (right).</p>
<p><strong>Provide a selected-state version of an interface icon only if necessary.</strong> You don’t need to provide selected and unselected appearances for an icon that’s used in standard system components such as toolbars, tab bars, and buttons. The system updates the visual appearance of the selected state automatically.</p>
<p><img src="https://docs-assets.developer.apple.com/published/5e11d32f07a319b9e0fa89ba33937617/icons-selection-correct~dark%402x.png" alt="An image of two toolbar buttons that share a background. The left button shows the Filter icon in a selected state, using a blue tint color for its background. The right button shows the More icon in an unselected state, using the default appearance for toolbar buttons."></p>
<p>In a toolbar, a selected icon receives the app’s accent color.</p>
<p><strong>Use inclusive images.</strong> Consider how your icons can be understandable and welcoming to everyone. Prefer depicting gender-neutral human figures and avoid images that might be hard to recognize across different cultures or languages. For guidance, see <a href="https://developer.apple.com/design/human-interface-guidelines/inclusion">Inclusion</a>.</p>
<p><strong>Include text in your design only when it’s essential for conveying meaning.</strong> For example, using a character in an interface icon that represents text formatting can be the most direct way to communicate the concept. If you need to display individual characters in your icon, be sure to localize them. If you need to suggest a passage of text, design an abstract representation of it, and include a flipped version of the icon to use when the context is right-to-left. For guidance, see <a href="https://developer.apple.com/design/human-interface-guidelines/right-to-left">Right to left</a>.</p>
<p><img src="https://docs-assets.developer.apple.com/published/2274cfee48a01e26313f5a3d56ccea0b/character-in-glyph~dark%402x.png" alt="A partial screenshot of the SF Symbols app showing the info panel for the character symbol, which looks like the capital letter A. Below the image, the following eight localized versions of the symbol are listed: Latin, Arabic, Hebrew, Hindi, Japanese, Korean, Thai, and Chinese."></p>
<p>Create localized versions of an icon that displays individual characters.</p>
<p><img src="https://docs-assets.developer.apple.com/published/e634b09c6eb7f3fe20896574528365dd/abstract-text-in-glyph~dark%402x.png" alt="A partial screenshot of the SF Symbols app showing the info panel for the text dot page symbol, which looks like three left-aligned horizontal lines inside a rounded rectangle. Below the image, the left-to-right and right-to-left localized versions are shown."></p>
<p>Create a flipped version of an icon that suggests reading direction.</p>
<p><strong>If you create a custom interface icon, use a vector format like PDF or SVG.</strong> The system automatically scales a vector-based interface icon for high-resolution displays, so you don’t need to provide high-resolution versions of it. In contrast, PNG — used for app icons and other images that include effects like shading, textures, and highlighting — doesn’t support scaling, so you have to supply multiple versions for each PNG-based interface icon. Alternatively, you can create a custom SF Symbol and specify a scale that ensures the symbol’s emphasis matches adjacent text. For guidance, see <a href="https://developer.apple.com/design/human-interface-guidelines/sf-symbols">SF Symbols</a>.</p>
<p><strong>Provide alternative text labels for custom interface icons.</strong> Alternative text labels — or accessibility descriptions — aren’t visible, but they let VoiceOver audibly describe what’s onscreen, simplifying navigation for people with visual disabilities. For guidance, see <a href="https://developer.apple.com/design/human-interface-guidelines/voiceover">VoiceOver</a>.</p>
<p><strong>Avoid using replicas of Apple hardware products.</strong> Hardware designs tend to change frequently and can make your interface icons and other content appear dated. If you must display Apple hardware, use only the images available in <a href="https://developer.apple.com/design/resources/">Apple Design Resources</a> or the SF Symbols that represent various Apple products.</p>

<div class="sl-heading-wrapper level-h4"><h4 id="document-icons">Document icons</h4><a class="sl-anchor-link" href="#document-icons"><span aria-hidden="true" class="sl-anchor-icon"><svg width="16" height="16" viewBox="0 0 24 24" fill="currentColor"><path d="m12.11 15.39-3.88 3.88a2.52 2.52 0 0 1-3.5 0 2.47 2.47 0 0 1 0-3.5l3.88-3.88a1 1 0 1 0-1.42-1.42l-3.88 3.89a4.48 4.48 0 0 0 6.33 6.33l3.89-3.88a1 1 0 0 0-1.42-1.42m8.58-12.08a4.49 4.49 0 0 0-6.33 0l-3.89 3.88a1 1 0 1 0 1.42 1.42l3.88-3.88a2.52 2.52 0 0 1 3.5 0 2.47 2.47 0 0 1 0 3.5l-3.88 3.88a1 1 0 0 0 0 1.42 1 1 0 0 0 1.42 0l3.88-3.89a4.49 4.49 0 0 0 0-6.33M8.83 15.17a1 1 0 0 0 .71.29 1 1 0 0 0 .71-.29l4.92-4.92a1 1 0 1 0-1.42-1.42l-4.92 4.92a1 1 0 0 0 0 1.42"></path></svg></span><span class="sr-only" data-pagefind-ignore>Section titled “Document icons”</span></a></div>
<p>If your macOS app can use a custom document type, you can create a document icon to represent it. Traditionally, a document icon looks like a piece of paper with its top-right corner folded down. This distinctive appearance helps people distinguish documents from apps and other content, even when icon sizes are small.</p>
<p>If you don’t supply a document icon for a file type you support, macOS creates one for you by compositing your app icon and the file’s extension onto the canvas. For example, Preview uses a system-generated document icon to represent JPG files.</p>
<p><img src="https://docs-assets.developer.apple.com/published/bfe462604c63811cb542e7c0fc46185e/doc-icon-generated%402x.png" alt="An image of the Preview document icon for a JPG file."></p>
<p>In some cases, it can make sense to create a set of document icons to represent a range of file types your app handles. For example, Xcode uses custom document icons to help people distinguish projects, AR objects, and Swift code files.</p>
<p><img src="https://docs-assets.developer.apple.com/published/8cd56a7291cd6b41fe391958f704c823/doc-icon-custom-1%402x.png" alt="Image of an Xcode project document icon."></p>
<p><img src="https://docs-assets.developer.apple.com/published/a1449177968f693c1bd68c2b146df7c3/doc-icon-custom-2%402x.png" alt="Image of a document icon for an AR object."></p>
<p><img src="https://docs-assets.developer.apple.com/published/495bd043bf65349ec96f6728941386f8/doc-icon-custom-3%402x.png" alt="Image of a document icon for a Swift file."></p>
<p>To create a custom document icon, you can supply any combination of background fill, center image, and text. The system layers, positions, and masks these elements as needed and composites them onto the familiar folded-corner icon shape.</p>
<p><img src="https://docs-assets.developer.apple.com/published/2aed446834a2dc6e8275b6bd7a797ca9/doc-icon-parts-background-fill%402x.png" alt="A square canvas that contains a grid of pink lines and a jagged white EKG line that runs horizontally across the middle. The pink grid gets lighter in color toward the bottom edge."></p>
<p>Background fill</p>
<p><img src="https://docs-assets.developer.apple.com/published/b59c836903d1b409ab9e21f81762df3e/doc-icon-parts-center-image%402x.png" alt="A solid pink heart."></p>
<p>Center image</p>
<p><img src="https://docs-assets.developer.apple.com/published/59fcacf0de76207baf14b3d21da6955a/doc-icon-parts-text~dark%402x.png" alt="The word heart in all caps."></p>
<p>Text</p>
<p><img src="https://docs-assets.developer.apple.com/published/d5da9148d27f60891780ab1a9546a111/doc-icon-parts%402x.png" alt="A custom document icon that displays the pink heart and the word heart on top of the pink grid and white EKG line."></p>
<p>macOS composites the elements you supply to produce your custom document icon.</p>
<p><a href="https://developer.apple.com/design/resources/#macos-apps">Apple Design Resources</a> provides a template you can use to create a custom background fill and center image for a document icon. As you use this template, follow the guidelines below.</p>
<p><strong>Design simple images that clearly communicate the document type.</strong> Whether you use a background fill, a center image, or both, prefer uncomplicated shapes and a reduced palette of distinct colors. Your document icon can display as small as 16x16 px, so you want to create designs that remain recognizable at every size.</p>
<p><strong>Designing a single, expressive image for the background fill can be a great way to help people understand and recognize a document type.</strong> For example, Xcode and TextEdit both use rich background images that don’t include a center image.</p>
<p><img src="https://docs-assets.developer.apple.com/published/8cd56a7291cd6b41fe391958f704c823/doc-icon-custom-1%402x.png" alt="Image of an Xcode project document icon."></p>
<p><img src="https://docs-assets.developer.apple.com/published/f32709a5ff5742e79fd03a58ae1dd9c6/doc-icon-fill-only%402x.png" alt="Image of a TextEdit rich text document icon."></p>
<p><strong>Consider reducing complexity in the small versions of your document icon.</strong> Icon details that are clear in large versions can look blurry and be hard to recognize in small versions. For example, to ensure that the grid lines in the custom heart document icon remain clear in intermediate sizes, you might use fewer lines and thicken them by aligning them to the reduced pixel grid. In the 16x16 px size, you might remove the lines altogether.</p>
<p><img src="https://docs-assets.developer.apple.com/published/1f8bc7946a75363224f373924b557a3a/doc-icon-fewer-details-1%402x.png" alt="Pixelated image of the heart document icon. The grid, the EKG line, the heart shape, and the word heart are visible but blurry."></p>
<p>The 32x32 px icon has fewer grid lines and a thicker EKG line.</p>
<p><img src="https://docs-assets.developer.apple.com/published/e46ac887801d9a16393948c3f2098715/doc-icon-fewer-details-2%402x.png" alt="Pixelated image of the heart document icon, in which only the blurry heart shape and EKG line are visible."></p>
<p>The 16x16 px @2x icon retains the EKG line but has no grid lines.</p>
<p><img src="https://docs-assets.developer.apple.com/published/fd0d2afcd76a9b25c1217ef2ffb1ad0e/doc-icon-fewer-details-3%402x.png" alt="Pixelated image of the heart document icon, in which only the blurry heart shape is visible."></p>
<p>The 16x16 px @1x icon has no EKG line and no grid lines.</p>
<p><strong>Avoid placing important content in the top-right corner of your background fill.</strong> The system automatically masks your image to fit the document icon shape and draws the white folded corner on top of the fill. Create a set of background images in the sizes listed below.</p>
<ul>
<li>512x512 px @1x, 1024x1024 px @2x</li>
<li>256x256 px @1x, 512x512 px @2x</li>
<li>128x128 px @1x, 256x256 px @2x</li>
<li>32x32 px @1x, 64x64 px @2x</li>
<li>16x16 px @1x, 32x32 px @2x</li>
</ul>
<p><strong>If a familiar object can convey a document’s type or its connection with your app, consider creating a center image that depicts it.</strong> Design a simple, unambiguous image that’s clear and recognizable at every size. The center image measures half the size of the overall document icon canvas. For example, to create a center image for a 32x32 px document icon, use an image canvas that measures 16x16 px. You can provide center images in the following sizes:</p>
<ul>
<li>256x256 px @1x, 512x512 px @2x</li>
<li>128x128 px @1x, 256x256 px @2x</li>
<li>32x32 px @1x, 64x64 px @2x</li>
<li>16x16 px @1x, 32x32 px @2x</li>
</ul>
<p><strong>Define a margin that measures about 10% of the image canvas and keep most of the image within it.</strong> Although parts of the image can extend into this margin for optical alignment, it’s best when the image occupies about 80% of the image canvas. For example, most of the center image in a 256x256 px canvas would fit in an area that measures 205x205 px.</p>
<p><img src="https://docs-assets.developer.apple.com/published/582e097f1ccf8d0dca92897a51288b13/doc-icon-parts-margins~dark%402x.png" alt="Diagram of the solid pink heart shape within blue margins that measure 10 percent of the canvas width."></p>
<p><strong>Specify a succinct term if it helps people understand your document type.</strong> By default, the system displays a document’s extension at the bottom edge of the document icon, but if the extension is unfamiliar you can supply a more descriptive term. For example, the document icon for a SceneKit scene file uses the term <em>scene</em> instead of the file extension <em>scn</em>. The system automatically scales the extension text to fit in the document icon, so be sure to use a term that’s short enough to be legible at small sizes. By default, the system capitalizes every letter in the text.</p>
<p><img src="https://docs-assets.developer.apple.com/published/3b4bb7de9edb5870d3a162aae8153163/doc-icon-custom-extension%402x.png" alt="Image of a SceneKit scene document icon."></p>
